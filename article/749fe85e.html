FAQ：

1、left <= right，循环条件中 left 为什么要小于等于 right？

​ <= 是根据下面的代码得出来的，假设当前数据为 0、1、2 我要找到值 target 为 2，left 和 right 刚开始是一定满足条件的，所以执行下去，求 mid， mid = (0 + 1) / 2 求出来为 1，然后 mid 下标的值 与 target 进行比较，发现小于 target，于是对左端进行修改 left = mid + 1，这个时候 left 为 2，而此时 right 也为 2，如果循环条件不包含 = 的情况，那就按照不存在处理了，所以需要加上等于的情况也要执行。

2、left = mid + 1、right = mid - 1，为什么修改左端mid要加一，修改右端要mid减一 ？

​ 既然执行到这个语句，说明 mid 作为下标的值 已经大于或者小于 target ，没必要再在下一个循环中出现了。如果不进行这种操作，代码也不调整则会出现死循环，假设当前数据为 0、1、2 我要查找的值 target 为 2，进入循环 mid 等于 1，然后 mid 下标的值 与 target 进行比较，发现小于 target，于是对左端进行修改 left = mid，这个时候 left 为 1，那就这样一直循环下去没有出口。
